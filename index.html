STARBUKCS

1인 프로젝트

목표

실제 스타벅스 홈페이지 수준에 근접한 결과물을 목표로 해당 프로젝트를 기획하고 진행하였습니다.

주요 기능 및 특징

  • swpier를 이용한 슬라이더 구현
  • JavaScript를 이용한 동적 사이트
  • GSAP의 scrollTrigger를 기능을 사용한 반응형 웹 사이트
  • 사용자 환경에 따라 유연하게 반응할 수 있도록 rem 단위를 사용하여 다양한 화면 크기를 고려한 레이아웃을 구성하였습니다.

사용 기술/Tool

항목을 누르면 자세히 볼 수 있습니다.
➡️ vite

개발 생산성과 성능 향상을 위해 Vite를 활용하여 실시간 반영 기능을 적용하고, 최종 빌드 시 경량화 및 최적화를 수행하였습니다.

➡️ GSAP

GSAP ScrollTrigger 기능을 활용하여 특정 스크롤 구간에 맞춰 애니메이션을 트리거하도록 구성하였습니다.

➡️ Swiper

사용자 친화적인 슬라이더 인터페이스 구현을 위해 Swiper 라이브러리를 도입하여 효과적인 콘텐츠 전환을 구현하였습니다.

➡️ Lighthouse

Lighthouse 분석 결과: 성능(90점), 접근성(86점), 권장사항(78점), 검색엔진 최적화(91점)

SEO 및 웹 표준 준수 여부를 확인하고, 전반적인 웹사이트 품질 향상을 위해 Lighthouse를 활용하였습니다.

➡️ Web Developer

Web Developer의 HTML 유효성 검사 기능을 사용하여, 마크업 구조를 웹 표준에 맞게 구성하였습니다.

목표

컬리(Karly)의 실제 웹사이트를 클론하여 UI 및 기능을 구현한 프로젝트입니다.

주요 기능 및 특징

  • 디바이스 해상도에 따라 UI가 동적으로 조정되어 일관된 사용자 경험을 제공하도록 설계하였습니다.
  • 모바일 환경에서도 원활하게 작동하도록 반응형으로 설계하였습니다.
  • 사용자 환경에 따라 유연하게 반응할 수 있도록 rem 단위를 사용하여 다양한 화면 크기를 고려한 레이아웃을 구성하였습니다.

➡️ Semantic HTML elements

웹 접근성과 유지보수성을 고려하여 HTML5의 시맨틱 태그를 적극 활용하였습니다.

디자인 시스템의 일관성과 유지보수 효율성을 높이기 위해 Tailwind CSS를 적용하였습니다.
